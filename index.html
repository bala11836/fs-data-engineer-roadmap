⭐ 1.1 What is Cloud Computing (Must Know)

Definition: Using computers, storage, and services over the internet instead of owning physical hardware
Analogy: Cloud = renting electricity from the grid vs running your own generator
Before Cloud (On-Premise): Companies bought their own servers, kept them in server rooms, hired people to maintain them — expensive, slow to scale, hardware failures
With Cloud: Rent servers from Microsoft/Amazon/Google, pay only for what you use, scale up/down instantly, provider handles maintenance
5 Key Benefits of Cloud: 1) Cost Savings — pay-as-you-go, 2) Scalability — add resources in minutes, 3) Reliability — data backed up across locations, 4) Speed — deploy in minutes, 5) Global Reach — data centers worldwide

⭐ 1.2 Cloud Service Models (Must Know)

IaaS (Infrastructure as a Service) — You rent raw infrastructure. Get VMs, storage, networking. You manage OS, apps, data. Example: Azure VMs, AWS EC2. Analogy: Renting an empty house — you bring furniture
Diagram
Provider gives you: [Server Hardware] [Network] [Storage]
You manage:         [OS] [Applications] [Data] [Security]
PaaS (Platform as a Service) — You get a platform to build on. Get runtime, dev tools, database. You manage code & data. Example: Azure App Service, Heroku. Analogy: Renting a furnished apartment — just bring your clothes
Diagram
Provider gives you: [Server] [OS] [Runtime] [Database]
You manage:         [Your Application Code] [Data]
SaaS (Software as a Service) — You just use ready software. Provider manages everything. Example: Microsoft 365, Gmail, Salesforce, Zoom. Analogy: Staying in a hotel — everything is provided
Diagram
Provider gives you: [Everything — complete working software]
You manage:         [Your data and settings]
Quick Comparison Table — memorize this:
FeatureIaaSPaaSSaaS
You manageOS + AppsApps onlyNothing (just use)
ControlMostSomeLeast
FlexibilityMostMediumLeast
Ease of useHardMediumEasy
ExampleAzure VMsAzure App ServiceMicrosoft 365

⭐ 1.3 Cloud Deployment Models (Must Know)

Public Cloud — Open to everyone, shared infrastructure. Azure, AWS, Google Cloud. Cheapest, most scalable, less control over security.
Private Cloud — Dedicated to one organization. More security, more control, more expensive.
Hybrid Cloud — Combination of public + private. Keep sensitive data on private, use public for scalable workloads. 🔥 Most common in Financial Services (banks use this!)
Multi-Cloud — Using multiple public clouds (Azure + AWS)

⭐ 1.4 What is Microsoft Azure (Must Know)

Definition: Microsoft's cloud computing platform with 200+ services and 60+ regions worldwide
Why Azure for your role: Accenture is a Microsoft partner, many FS clients use Azure, Palantir Foundry often deploys on Azure, Azure data services connect to Foundry
Azure Portal: portal.azure.com — web interface to manage everything
Azure Account Structure:
Structure
Azure Account (your login)
  └── Subscription (billing boundary)
      └── Resource Group (container for related resources)
          └── Resources (VMs, databases, storage, etc.)

🌐 1.5 Azure Global Infrastructure

Regions: Physical locations with data centers. India has 3: Central India (Pune), South India (Chennai), West India (Mumbai)
Availability Zones: Separate buildings within a region for disaster protection. Minimum 3 zones per region. If one goes down, others keep running.
Region Pairs: Each region paired with another 300+ km away. If entire region fails, paired region takes over. Example: Central India ↔ South India

⭐ 2.1 Azure Compute Services (Must Know)

Azure Virtual Machines (VMs) — A computer in the cloud (choose OS, CPU, RAM, storage). Need full control over OS. Pay per hour/minute.
Example
Example: You need a Linux server to run a data processing script
→ Create a VM with Ubuntu OS, 16 GB RAM, 4 CPUs
→ Install Python, PySpark on it
→ Run your script
→ Delete VM when done (stop paying)
Azure App Service — PaaS for hosting web apps, APIs. No need to manage VMs or OS. Supports Python, Java, .NET, Node.js.
Azure Functions (Serverless) — Run small pieces of code without managing servers. Pay only when code runs.
Example
File uploaded to Blob Storage
  → Triggers Azure Function
  → Function reads file, processes it
  → Writes output to database
  → You pay only for the 5 seconds it ran
Azure Container Instances (ACI) & AKS — ACI: run single containers quickly. AKS: manage many containers at scale.
Azure Virtual Desktop — A Windows desktop in the cloud for remote work.

💡 2.2 Azure Networking Services

Azure Virtual Network (VNet) — Your own private network in Azure. Resources in same VNet can communicate.
Subnets — Smaller sections within a VNet to organize and isolate resources.
Network Security Groups (NSG) — Firewall rules: control which traffic is allowed IN and OUT.
Azure VPN Gateway — Secure tunnel between on-premise network and Azure.
Azure ExpressRoute — Private, dedicated connection to Azure (not over public internet). 🔥 Used by banks and financial institutions!
Azure Load Balancer — Distributes incoming traffic across multiple servers.
Azure DNS — Translates domain names to IP addresses.
Azure CDN — Caches content at edge locations close to users for faster loading.

⭐🔥 2.3 Azure Storage Services (Critical)

Azure Storage Account — Container for all storage services. Globally unique name. Redundancy options: LRS (3 copies same DC), ZRS (3 copies across zones), GRS (6 copies across 2 regions), RA-GRS (GRS + read from secondary)
Azure Blob Storage — Store any file type (Binary Large OBject). Access Tiers: Hot, Cool, Cold, Archive.
Structure
Storage Account
  └── Container (like a folder)
      └── Blobs (your files)
          ├── data.csv
          ├── photo.jpg
          └── backup.sql
🔥🔥🔥 Azure Data Lake Storage Gen2 (ADLS Gen2) — Blob Storage + Hierarchical Namespace = perfect for big data. Palantir Foundry stores its data on ADLS Gen2!
Structure
ADLS Gen2 account
  └── filesystem (container)
      └── folder1/
          └── subfolder/
              ├── data_2026_01.parquet
              ├── data_2026_02.parquet
              └── data_2026_03.parquet
Azure Files — Managed file shares (SMB/NFS protocol), replace on-premise file servers.
Azure Queue Storage — Store messages in a queue for processing later.
Azure Table Storage — NoSQL key-value store for simple structured data.
Azure Disk Storage — Disks for VMs. Types: Ultra, Premium SSD, Standard SSD, Standard HDD.

⭐ 2.4 Azure Database Services

Azure SQL Database — Managed relational database (SQL Server in cloud). PaaS — Microsoft handles backups, updates, patches.
Azure Cosmos DB — Globally distributed NoSQL database. Supports Document, key-value, graph. APIs: SQL, MongoDB, Cassandra, Gremlin, Table.
Azure Database for MySQL / PostgreSQL — Managed open-source databases.
Azure Synapse Analytics — Data warehouse + analytics in one platform. 🔥 Important for Data Engineering!
Azure Cache for Redis — In-memory cache for fast data access.

⭐ GROUP BY & HAVING (1.5 hours)

GROUP BY: group rows and aggregate
SQL
SELECT department, AVG(salary) AS avg_salary
FROM employees
GROUP BY department;

SELECT department, COUNT(*) AS emp_count,
       AVG(salary) AS avg_salary,
       MIN(salary) AS min_salary,
       MAX(salary) AS max_salary
FROM employees
GROUP BY department;
HAVING: filter AFTER grouping
SQL
SELECT department, COUNT(*) AS emp_count
FROM employees
GROUP BY department
HAVING COUNT(*) > 5;
WHERE vs HAVING: WHERE filters BEFORE grouping, HAVING filters AFTER grouping

🔥🔥🔥 JOINs — THE MOST IMPORTANT CONCEPT (2.5 hours)

INNER JOIN — only matching rows from both tables
LEFT JOIN — all from left + matching from right
RIGHT JOIN — all from right + matching from left
FULL OUTER JOIN — all from both tables
CROSS JOIN — every combination (cartesian product)
SELF JOIN — table joined with itself (e.g., employee & manager)
SQL
-- INNER JOIN
SELECT e.first_name, d.department_name
FROM employees e
INNER JOIN departments d ON e.department_id = d.id;

-- LEFT JOIN
SELECT e.first_name, d.department_name
FROM employees e
LEFT JOIN departments d ON e.department_id = d.id;

-- SELF JOIN: employee's manager
SELECT e.first_name AS employee, m.first_name AS manager
FROM employees e
LEFT JOIN employees m ON e.manager_id = m.id;

-- Joining 3+ tables
SELECT e.first_name, d.department_name, l.city
FROM employees e
JOIN departments d ON e.department_id = d.id
JOIN locations l ON d.location_id = l.id;

🔥🔥🔥 Window Functions — CRITICAL FOR DATA ENGINEERING (2 hours)

ROW_NUMBER() — unique sequence
RANK() vs DENSE_RANK() — RANK skips, DENSE_RANK doesn't
SQL
SELECT first_name, department, salary,
       ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) AS rank
FROM employees;

-- RANK: 1, 2, 2, 4 (skips 3)
-- DENSE_RANK: 1, 2, 2, 3 (no skip)
LAG() / LEAD() — access previous/next row values
SQL
SELECT transaction_date, amount,
       LAG(amount) OVER (ORDER BY transaction_date) AS prev_amount,
       amount - LAG(amount) OVER (ORDER BY transaction_date) AS change
FROM transactions;
Running Total and Moving Average
SQL
-- Running Total
SELECT transaction_date, amount,
       SUM(amount) OVER (ORDER BY transaction_date) AS running_total
FROM transactions;

-- Moving Average (3-period)
SELECT transaction_date, amount,
       AVG(amount) OVER (ORDER BY transaction_date
                         ROWS BETWEEN 2 PRECEDING AND CURRENT ROW) AS moving_avg_3
FROM transactions;
FIRST_VALUE / LAST_VALUE / NTH_VALUE / NTILE(n)

# Decorators 🔥
def my_decorator(func):
    def wrapper(*args, **kwargs):
        print("Before function call")
        result = func(*args, **kwargs)
        print("After function call")
        return result
    return wrapper

@my_decorator
def say_hello(name):
    print(f"Hello {name}")
